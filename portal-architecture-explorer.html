Portal Pages & Features
Every module in the current production portal

Dashboard

KPI summary cards, bar/pie/radar charts via Recharts, project overview table, and a Today's Schedule widget fed live from the BD follow-up queue showing due and overdue calls.

Analytics

Clients Map

Interactive Leaflet.js map clustering clients by state. Bubble radius scales by total requirement_kwh. Popup shows per-client capacity and cumulative state total in kWh or MWh.

Geo / Map

Projects

Project cards linked to clients. Each project can be linked to a commercial proposal via inline picker (filtered by client). If no proposal exists, a "New Proposal" shortcut routes to the Configurator.

Core

BD Pipeline

Full Kanban board across 6 BD stages. Cards show company, capacity, and assigned owner. Move buttons trigger PATCH to update stage. Error banners surface failures inline.

Sales CRM

Follow-up Queue

Prioritised follow-up list segmented into Overdue, Due Today, This Week, Later, and Done. Per-card actions: Log Activity, Mark Done, Snooze 2d. Theme-aware with full dark-mode support.

BD Ops

Proposals

Commercial proposals generated from the Configurator. Linked to both a client_id (on proposals) and a proposal_id (on projects). Shows CAPEX, IRR, and payback on linked project cards.

Commercial

Configurator

System sizing tool for building UnityESS quotes. Selects number of 125 kW / 261 kWh units, computes ToD arbitrage savings, and generates a linked proposal record in the database.

Quoting

BD Accounts & Contacts

Company-level accounts with associated contacts. Each opportunity is linked to an account. Contacts store role, phone, and email for outreach logging.

Security
Authentication & RBAC
Three-tier auth model: open BD routes, auth-gated import, admin-only automation

Open Routes — BD CRUD

All BD read/write routes (/api/bd/accounts, /api/bd/opportunities, /api/bd/follow-ups, etc.) run without any authentication. This is intentional for an internal tool where all users are trusted.

No middleware All BD CRUD

Auth-Gated — Import

Bulk import endpoints (/api/bd/import/*) require a valid JWT in the Authorization: Bearer … header. Any logged-in user can import. The requireAuth middleware verifies the token against JWT_SECRET.

requireAuth Any logged-in user

Admin-Only — Automation

/api/bd/automation/run requires both a valid JWT and role === 'admin'. This is enforced by requireAuth + requireRole('admin') chained middleware.

requireAuth requireRole('admin')

JWT Token Flow

POST /api/auth/login validates email + bcrypt password hash, then signs a JWT with { id, email, role } payload. The token is stored client-side and sent as a Bearer header on protected calls.

HS256 / JWT_SECRET Stateless

Bugs & Resolutions
Issues Encountered & Fixed
All non-trivial bugs hit during development and how they were resolved

Kanban Move Buttons — Silent 401

PATCH /api/bd/opportunities/:id was blocked by requireRole('admin','bd_exec'). The catch (_) {} block swallowed all errors silently. Users saw nothing happen on click.

Fix: Stripped role requirement from all BD CRUD routes. Added movingId + stageErr state with dismissible error banner on the Kanban board.

✓ Fixed

JWT_SECRET Mismatch in Production

Login tokens were signed with a local dev JWT_SECRET. Production Vercel used a different env var value, causing all requireAuth-protected BD routes to return 401 in prod even with a valid token.

Fix: Removed requireAuth from all BD CRUD routes. Only import and automation endpoints retain auth.

✓ Fixed

Follow-up Cards White in Dark Mode

Cards used hardcoded light pastels (#fecaca, #fde68a, #e8f4ff, #fafafa, #f0f0f0). The portal sidebar uses a hardcoded dark background, making white cards visually jarring.

Fix: Replaced all hardcoded colours with hsl(var(--card)), hsl(var(--border)), hsl(var(--muted)), rgba() opacity-based borders.

✓ Fixed

git index.lock on macOS Mount

The working directory is a macOS-mounted FS. git add / git commit always fail with index.lock: File exists because the lock file cannot be removed by the git process.

Fix: Git plumbing workflow — GIT_INDEX_FILE=/tmp/idx git read-tree → hash-object -w → update-index → write-tree → commit-tree → push.

✓ Fixed (workaround)

"BD Pipeline" Nav Label Not Deployed

Layout.jsx had label: 'BD Pipeline' locally but the change lived on a feature branch, not on main. Production showed the old label.

Fix: Pushed Layout.jsx directly to main via git plumbing. Vercel auto-deployed on push.

✓ Fixed

Client Map — Count vs. Capacity Bubbles

Map bubbles originally showed client count per state cluster. This was not meaningful for a BESS sales tool — capacity in kWh/MWh is the relevant signal.

Fix: Bubble radius now scales with total requirement_kwh per state. Label shows MWh if ≥1000 kWh, otherwise kWh. Falls back to count if no capacity data.

✓ Improved
