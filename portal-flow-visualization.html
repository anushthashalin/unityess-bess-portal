BESS Configurator — Sizing Engine
LFP battery storage sizing for C&I. Two use cases. Rule-based engine with degradation-adjusted 10-year finance model.
Use Case Flow
DG Replacement
Displaced diesel savings
or
ToD Arbitrage
Peak / off-peak spread
// DG Replacement raw_kwh = load_kw × backup_hrs net_benefit = (dg_eff × fuel_₹) − 3 // ₹3/kWh grid charge // ToD Arbitrage raw_kwh = dispatch_kwh / 0.85 // AC loss pre-derate ann_savings = dispatch × (peak−off-peak) × days // Derating (both cases) validated = raw_kwh / (1 − 0.15) // 5% AC + 10% SOC
SKU Matching — Two Configurations
ECONOMICAL
ceil(validated ÷ sku.kWh)
RECOMMENDED
+15% uplift · dispatch factor
dispatch_factor = soh_yr1 × rte_yr1 × 0.90 ≈ 0.95 × 0.97 × 0.90 ≈ 0.83 rec_units = ceil(validated × 1.15 / (sku.kwh × dispatch_factor))

Year-by-Year Degradation Formula
at_meter[N] = nameplate × soh[N] × 0.90 × rte[N] gross[N] = at_meter[N] × tariffDiff × cycles_yr om[N] = capex × 1.5% × 1.03^(N−1) // +3%/yr net_cf[N] = gross[N] − om[N]

EPC Solar Configurator
Rooftop and ground-mount solar PV sizing with 25-year financial model. Mirrors BESS track with a separate product_type filter on BD pipeline.
Three Sizing Modes
Load-Based
Monthly kWh → kWp via CUF × self-consumption %
Area-Based
Roof area (sq m) → kWp via m²/kWp (module Wp)
Manual
Direct kWp entry — user specifies system size
// Load-Based sizing kWp = ceil((monthly_kwh / (cuf × 30.5 × 24)) / (self_consump% / 100)) // Area-Based sizing sqm_per_kwp = (1000/module_wp) × 1.7 // ~550Wp modules kWp = area_sqm / sqm_per_kwp
Generation Model
// Annual generation annualGenKwh = systemKwpDC × cuf × 8760 // Self-consumed vs exported selfConsumed = annualGenKwh × (selfConsump/100) exported = annualGenKwh − selfConsumed // Year-1 savings savings_yr1 = selfConsumed × gridTariff + exported × gridTariff × 0.70 // net metering

Known Issues & Backlog
From PORTAL_AUDIT_REPORT.md, March 23 2026. Sorted by production impact.
P0 — Block Production
P0Container SKU pricing = ₹0
6 container SKUs have no price in DB. Finance model shows ₹0 CAPEX, IRR, NPV for all container configurations.
P0All write endpoints lack authentication
POST/PATCH for accounts, contacts, proposals, clients — no requireAuth() call. Unauthenticated users can modify all data.
P0Password hashes in GET /api/bd/users
bcrypt hashes returned in response body. Must exclude password field from all user GET responses.
P0Null/FK violations crash API with 500
No input validation before DB operations. Missing required fields and invalid FKs cause unhandled 500 crashes instead of 400 responses.
P2 — Fix This Sprint
P2Assumptions hardcoded (SOC, O&M, uplift, escalation)
SOC window, O&M rate, uplift 15%, escalation 3% — all hardcoded JS. Require code deploy to change. Should be in DB as configurable master.
P2Cycle datasets hardcoded in frontend
All three LFP cycle datasets (SOH/RTE tables) are hardcoded JS constants. Not editable without a deploy.
P2SKU price snapshot not fully immutable
Prices at save-time are captured in the snapshot, but master price changes are not blocked from affecting historical records.
P3 — Next Quarter
P3BESS Sizing ↔ BD Opportunity linkage
No bridge between sizing_analyses and bd.opportunities. Requires a join table and UI on the opportunity detail page.
P3EPC Sizing ↔ BD Opportunity linkage
Same gap exists for the EPC solar configurator — no way to attach a solar sizing to an EPC deal.
P3Workflow stage machine for Projects
Projects table has no stage field, no transition validation, no history tracking. Both BESS and EPC affected.
